# 스마트 레시피 파인더 (Smart Recipe Finder) - Blueprint

## 1. 개요 (Overview)
사용자가 보유한 재료와 선호하는 요리 분야(한식, 중식, 일식, 양식)를 입력하면, 해당 조건에 맞는 최적의 레시피를 추천해주는 웹 애플리케이션입니다. 프레임워크 없이 순수 웹 표준 기술(Web Components, Modern CSS)로 구현됩니다.

## 2. 프로젝트 상세 (Project Outline)

### 스타일 및 디자인 (Style & Design)
- **테마:** 신선함과 식욕을 돋우는 '프레시 가든(Fresh Garden)' 테마. (주요 색상: OKLCH 기반의 선명한 그린, 웜 오렌지, 크림 화이트)
- **UI 구성 요소:**
  - **카테고리 카드:** 요리 분야를 선택할 수 있는 인터랙티브한 카드.
  - **재료 칩(Chips):** 입력한 재료를 시각적으로 관리하는 태그 시스템.
  - **레시피 카드:** 요리 사진, 소요 시간, 난이도, 일치하는 재료 수를 표시.
- **시각 효과:** 
  - 유리창 효과(Glassmorphism)를 적용한 모달 및 카드.
  - 부드러운 호버 애니메이션 및 전환 효과.
  - 배경에 미세한 노이즈 텍스처를 적용하여 고급스러운 느낌 부여.

### 주요 기능 (Features)
- **분야 선택:** 한식, 일식, 중식, 양식 중 택일.
- **재료 관리:** 사용자가 입력하는 재료를 실시간으로 태그로 변환 및 삭제 기능.
- **필터링 검색:** 선택한 분야 내에서 입력된 재료가 가장 많이 포함된 순서대로 레시피 정렬.
- **레시피 상세 보기:** 요리 순서 및 팁을 제공하는 상세 뷰.

## 3. 현재 구현 계획 (Current Plan)

### 1단계: 기본 구조 및 스타일 설정
- `index.html`: 기본 레이아웃 및 웹 컴포넌트 구조 설정.
- `style.css`: 현대적인 CSS (Container Queries, Layers, Variables) 적용.
- `main.js`: 데이터 모델(레시피 샘플 데이터) 및 상태 관리 로직 초기화.

### 2단계: 핵심 컴포넌트 개발
- `<cuisine-selector>`: 요리 분야 선택 컴포넌트.
- `<ingredient-input>`: 재료 입력 및 태그 관리 컴포넌트.
- `<recipe-list>`: 필터링된 결과를 보여주는 리스트 컴포넌트.

### 3단계: 검색 로직 및 인터랙션 구현
- 선택된 카테고리와 재료를 기반으로 한 필터링 알고리즘 구현.
- 각 단계 간의 부드러운 화면 전환(Step-by-step UI).
